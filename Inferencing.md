# Full LLM inference stack on local Kubernetes

**Stack: vLLM Semantic Router вҶ’ Gateway (Istio Gateway using Gateway API implementation of Kubernetes with Inference Extension EPP) вҶ’ llm-d ModelService вҶ’ vLLM + DRA**

- Single Envoy-based gateway stack.
- Gateway is both the traffic proxy AND llm-d's gateway provider.
- The Inference Gateway Extension (EPP) plugs into the same Envoy proxy for KV-cache-aware routing to vLLM pods.

## What's included

| Component | Runs where | Purpose |
|---|---|---|
| **Gateway** | Inside K8s | Gateway API implementation of Istio |
| **llm-d EPP** | Inside K8s | Inference scheduler вҖ” smart pod-level routing |
| **Inference Simulator** | Inside K8s | Fake vLLM вҖ” tests orchestration without GPUs |
| **DRA manifests** | Inside K8s | DeviceClass + ResourceClaimTemplate вҖ” ready for real GPUs |
| **Semantic Router** | Inside K8s | Intent classifier вҖ” routes to best-fit model |
| **vllm-metal** | Native macOS | Real inference on Apple Metal GPU |
